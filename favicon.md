# Favicon

`favicon.svg` is the site icon: a hand-drawn-style golden spiral on a brown
tile, using the site palette (`#7B3A0E` background, `#E8D840` spiral). It's an
inline SVG referenced from `index.html` with:

```html
<link rel="icon" href="favicon.svg" type="image/svg+xml">
```

There's no PNG or `apple-touch-icon` fallback. SVG favicons work in all current
browsers. Safari caches favicons hard and is flaky on `localhost`, so test icon
changes on the deployed URL (or a Private window) rather than local dev.

## Regenerating it

The spiral path is generated by the script below, not hand-edited. It's an
Archimedean spiral (radius grows with angle) with a deterministic wobble added
so the line looks drawn rather than plotted. The script computes the path,
auto-centers it in the 64x64 tile, and writes `favicon.svg`.

Save the script below as `generate_favicon.py` and run it from the repo root
(or paste it straight into `python3 - <<'PY' ... PY`):

```python
import math

# --- knobs ---------------------------------------------------------------
TURNS     = 1.9      # number of loops. lower = looser, fewer arms
GROWTH    = 1.95     # radius gained per radian. higher = wider gaps
START_R   = 2.5      # radius at the center curl
DIRECTION = -1       # -1 or +1 to flip which way it winds
WOBBLE    = 0.07     # hand-drawn wander, as a fraction of radius (0 = clean)
STROKE    = 4.2      # line thickness
BG, FG    = "#7B3A0E", "#E8D840"
SIZE      = 64
# -------------------------------------------------------------------------

cx = cy = SIZE / 2
pts = []
theta = 0.0
tmax = TURNS * 2 * math.pi
while theta <= tmax:
    base = START_R + GROWTH * theta
    amp = WOBBLE * base + 0.3
    wob = amp * (math.sin(theta * 1.9 + 0.6) + 0.5 * math.sin(theta * 3.5 + 2.4))
    r = base + wob
    ang = DIRECTION * theta + 0.05 * math.sin(theta * 2.5)
    pts.append((cx + r * math.cos(ang), cy + r * math.sin(ang)))
    theta += 0.16

# auto-center the spiral's bounding box in the tile
xs = [p[0] for p in pts]
ys = [p[1] for p in pts]
dx = cx - (min(xs) + max(xs)) / 2
dy = cy - (min(ys) + max(ys)) / 2

d = "M " + " L ".join(f"{round(x, 2)} {round(y, 2)}" for x, y in pts)

svg = f'''<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 {SIZE} {SIZE}" role="img" aria-label="Spiralpalooza">
  <rect width="{SIZE}" height="{SIZE}" rx="12" fill="{BG}"/>
  <g transform="translate({round(dx, 2)},{round(dy, 2)})">
    <path d="{d}"
          fill="none" stroke="{FG}" stroke-width="{STROKE}" stroke-linecap="round" stroke-linejoin="round"/>
  </g>
</svg>
'''

with open("favicon.svg", "w") as f:
    f.write(svg)
print("wrote favicon.svg")
```

## Tuning notes

- **Too tight / too many arms?** Lower `TURNS` and/or raise `GROWTH`.
- **Wrong direction?** Flip `DIRECTION` between `-1` and `1`.
- **Too perfect/mechanical?** Raise `WOBBLE`; set it to `0` for a clean spiral.
- **Line too thin/thick?** Adjust `STROKE`.
- **Colors** come from `BG`/`FG`; keep them matched to the `--cf-*` palette in
  `index.html`.

After regenerating, eyeball `favicon.svg` in a browser, then commit both it and
any `index.html` change together.
